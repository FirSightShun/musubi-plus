📖 1. 背景：Bradley-Terry 损失与 Reward Model

理解 BT 模型在偏好学习中的核心地位

Bradley-Terry 模型原理

BT 模型将偏好比较建模为概率分布：给定两个输出 yw（chosen）和 yl（rejected），yw 被偏好的概率为：

BT 偏好概率 P(yw ≻ yl | x) = σ(r(x, yw) − r(x, yl)) = 1 / (1 + exp(r(x, yl) − r(x, yw)))

其中 r(·) 为 Reward Model 输出的标量分数，σ 为 sigmoid 函数。训练目标为最大化观测偏好数据的对数似然：

BT 损失函数（负对数似然） LBT = −𝔼(x,yw,yl)∼D [ log σ(r(x, yw) − r(x, yl)) ]

核心思想：chosen 的奖励分数应高于 rejected，两者差值越大，损失越小。

在图像 Edit 场景的适配

对于图像编辑任务，BT 损失的输入为三元组 (Isrc, P, Iedit)：

LBT = −𝔼 [ log σ(r(Isrc, P, Iedit,w) − r(Isrc, P, Iedit,l)) ]
Isrc源图像（编辑前）
P文本编辑指令（如 "将背景改为蓝天"）
Iedit,wchosen 编辑结果（人类偏好更高）
Iedit,lrejected 编辑结果（人类偏好更低）

💡 BT 损失 vs 其他偏好学习方法的定位

BT 损失是 Reward Model 训练的标准选择，输出标量奖励分数，可服务于后续 PPO 训练或数据筛选。DPO 虽然也基于 BT 模型，但它绕过了显式 Reward Model 训练，直接优化策略。选择 BT 训练 Reward Model 的核心价值在于：获得可复用的评分函数，用于数据筛选、在线 RL、模型评估等多场景。

⚡ 5. 方案三：SPIE（RLAIF + DPO，绕过显式 RM）

在线自对弈训练，无需显式 Reward Model，极低资源需求

SPIE ICCV 2025W

论文：SPIE: Semantic and Structural Post-Training of Image Editing Diffusion Models（arXiv:2504.12833）

核心思路：DPO 绕过 Reward Model

虽然基于 BT 偏好模型，但通过 DPO 数学变换，将 BT 中的奖励函数表达为策略与参考策略的比值，从而消除了训练显式 Reward Model 的需要：

DPO 损失（隐式 BT） LDPO = −𝔼 [ log σ(β log πθ(yw|x)/πref(yw|x) − β log πθ(yl|x)/πref(yl|x)) ]

AI 反馈替代人类标注

评分组件模型评分维度
结构评分Depth Anything V2输入-编辑图像深度图 L1 距离
语义评分DreamSim + grounded-SAM2编辑区域语义对齐 + 非编辑区保持
提示编码CLIPCross-attention 条件注入

极简训练需求

参考图像5 张/概念
训练步数仅 10 步
人类标注零（完全 RLAIF）
基础架构InstructPix2Pix (SD v1.5)
✅ 优势
  • 无需训练显式 Reward Model
  • 无需人工标注，零标注成本
  • 训练极快（10 步），资源需求极低
  • 在线训练，不受离线数据分布限制
  • 结构 + 语义双维度对齐
⚠️ 局限
  • 无显式 Reward Model，无法作为 Critic 复用
  • AI 反馈质量上限受限于评分组件能力
  • 基于 SD v1.5，对现代 DiT 架构适配待验证
  • 无法输出标量分数用于数据筛选

⚠️ 关键提示

如果你的目标是获得一个可复用的 Critic Model（Reward Model），SPIE 方案不适用——它通过 DPO 绕过了 RM 训练。但如果你只需要对齐编辑模型本身，SPIE 是资源最友好的路线。

🔀 7. 方案五：Classification Reward Model

BT RM 的替代方案——将偏好判断转为二分类任务

Classification RM 原理

将 (chosen, rejected) 对作为输入，输出 chosen 被偏好的概率（二分类）：

分类损失 LCLS = −[y · log p(chosen) + (1−y) · log(1 − p(chosen))]

与 BT RM 的核心区别：

维度BT Reward ModelClassification RM
输出标量奖励分数 r(x,y)偏好概率 p(chosen)
输入单个 (x,y) 对(x, yw, yl) 三元组
损失BT 负对数似然交叉熵
后续用途PPO 训练、数据筛选偏好判断、排序
灵活性可独立评分任意样本需成对比较

💡 实验结论（来自知乎深度分析）

在大多数实验中，Classification RM 可以取得不输于 BT RM 的性能，且实现更简单。但 Classification RM 无法输出独立标量分数，不适合作为 PPO 的 Critic Model。如果你的目标是训练一个可独立评分的 Reward Model，BT 方案仍是正确选择。

📦 8. 数据准备详解

从零构建偏好数据集的完整 Pipeline

数据准备通用 Pipeline

收集源数据

收集 (源图像, 编辑指令) 对。来源：公开基准（EmuEdit, MagicBrush, AnyEdit 等）+ 内部数据。EditReward 使用了 9,557 个指令-图像对作为种子。

多模型生成候选

用多个 SOTA 编辑模型（如 Step1X-Edit, Flux-Kontext, Qwen-Image-Edit, BAGEL, Ovis-U1, OmniGen2）对每个 (Isrc, P) 生成多个候选编辑结果。EditReward 每对生成 12 个候选。

人工偏好标注

对候选进行多维度人工评分。EditReward 采用 4-point Likert 量表（1=Poor, 4=Excellent），标注 2 个维度：指令遵循 + 视觉质量。每对评估 7 个候选，标注者间一致性 α≈0.67。

构建偏好对

根据标注分数，将同一指令下的候选两两比较，构建 (chosen, rejected) 偏好对。EditReward 最终得到 ~200K 偏好对。

数据质量控制

① 过滤标注者间一致性低的数据；② 处理平局样本（EditReward 的 Disentangling Ties 策略）；③ 多维度交叉验证。

🎯 自建数据集的规模建议

最小可行规模：10K~20K 偏好对（覆盖核心编辑类型）
推荐规模：50K~100K 偏好对（接近 ImageReward 水平）
理想规模：200K+ 偏好对（EditReward 水平，SOTA 性能）

关键发现（EditReward 验证）：数据质量远比数量重要——用 RM 筛选的 Top 20K 数据训练效果超过全量 46K 数据。

低成本数据构建策略

策略 A：利用开源数据集

直接使用 EditReward-Data（200K）或 ImageRewardDB（137K）作为基础，针对你的编辑模型补充少量领域数据（5K~10K）。

💰 成本：最低 | ⏱️ 周期：1~2 周

策略 B：AI 标注 + 人工校验

用 GPT-4o/VLM 进行初筛标注，仅对不确定样本进行人工校验。可大幅降低标注成本（估计降低 70%~80%）。

💰 成本：中等 | ⏱️ 周期：2~3 周

策略 C：全人工标注

最高质量保证。按 EditReward 流程，标注 200K 对需要约 3~5 名标注者 × 2~3 周。4-point Likert + 双维度。

💰 成本：最高 | ⏱️ 周期：3~5 周

框架详细说明

HuggingFace TRL

RewardTrainer 提供开箱即用的 RM 训练，内置 BT 损失函数。对语言模型支持完善，视觉模型需额外适配（自定义 collator + 模型 wrapper）。

适合：快速验证 BT 损失 + RM 训练的 baseline

OpenRLHF

v0.10 新增 VLM RLHF 支持（--data.image_key），Ray + vLLM 分布式架构可将 Actor/Reward/Critic 分离到不同 GPU，支持 70B+ 模型训练，支持 PPO/GRPO/DPO 等多种算法。

适合：生产级 VLM Reward Model 训练

自建方案

参考 EditReward 开源代码，基于 PyTorch + HuggingFace Transformers 手动实现。优势：可自由定制多维度损失、不确定性感知等高级特性。EditReward 代码库即采用此路线。

适合：需要 EditReward 级别定制化的场景

资源缩减策略

🔧 LoRA / QLoRA

仅微调 0.1%~1% 参数，显存需求降至全量的 1/4~1/5。EditReward 消融实验未测试 LoRA，但理论上可行。

GPU: 8×A800 → 2~4×A100

🔧 梯度检查点

用计算换显存，重计算前向激活值。可节省 60%~70% 激活显存，训练速度降低 ~20%。

显存节省: ~30%~40%

🔧 DeepSpeed ZeRO-3

将参数、梯度、优化器状态分片到多卡。配合 CPU offload 可进一步降低单卡显存需求。

适合: 4~8 卡分布式训练

⚠️ 关键风险提示

全量微调 7B VLM 的显存需求远超单卡（~120GB+），必须使用多卡 + ZeRO-3 或 LoRA 方案。
视觉输入的序列长度远超纯文本（图像 token 数通常 576~2048），激活显存占用显著更高。
Batch Size 受限：VLM 输入序列长 → 单卡 batch size 通常仅 1~2 → 需梯度累积模拟大 batch。

损失函数选择决策

标准 BT 损失

最经典、实现最简单。适合首次尝试、快速验证。TRL/OpenRLHF 原生支持。

推荐：入门 + baseline

多维度不确定性排序损失

EditReward 提出的 BT 扩展。考虑多维度 + 标注不确定性，性能最优但实现复杂。

推荐：追求 SOTA

DPO 隐式 BT

绕过 RM 训练，直接优化策略。无需显式 RM，但无法复用评分函数。

推荐：无需独立 RM 时

🏁 12. 总结与推荐路线

推荐实施路线（3 阶段）

Phase 1：Baseline 搭建（1~2 周）

① 使用 ImageReward 预训练模型作为初始 Critic，验证 RM-in-the-loop 训练流程
② 用 TRL RewardTrainer + 标准 BT 损失，在 EditReward-Data 子集（20K）上微调
③ 资源：2×A100-80G + LoRA，成本 < ¥3,000

Phase 2：编辑专用 RM 训练（2~3 周）

① 基于 EditReward 架构（Qwen2.5-VL-7B + 多维度 MLP Head）
② 使用 EditReward-Data 全量 200K 或自建数据（开源 + 领域补充）
③ 训练多维度不确定性感知排序损失（BT 变体）
④ 资源：8×A800 全量微调 或 4×A100 + LoRA，成本 ¥8,000~30,000

Phase 3：RM 驱动的编辑模型对齐（2~4 周）

① 数据筛选：用 RM 从大规模数据中筛选高质量子集（参考 EditReward: Top 20K > 全量 46K）
② 在线 RL：用 OpenRLHF 的 PPO/GRPO，以 RM 为 Reward 信号训练编辑模型
③ 持续迭代：RM 评估 → 发现 Bad Case → 补充数据 → 重新训练 RM

✅ 核心结论

1. 技术方案：EditReward（多维度 BT 变体）是当前图像编辑 RM 的 SOTA，推荐作为主力方案
2. 数据准备：200K 偏好对是理想规模；可从开源数据集起步，用 AI+人工混合标注降低成本；数据质量 > 数据数量
3. 训练框架：OpenRLHF（生产级 VLM 支持）或直接基于 EditReward 代码库
4. 训练资源：7B VLM 全量微调需 8×A800；LoRA 可降至 2~4×A100；成本 ¥3,000~30,000/月
5. 资源受限替代：3B 骨干 + LoRA + 20K 数据，2×A100 即可训练

⚠️ 风险提示

1. VLM 全量微调显存需求大，务必提前验证显存是否充足
2. 人工标注质量直接影响 RM 性能，标注者间一致性 α < 0.5 的数据建议丢弃
3. RM 可能存在 reward hacking 风险，需在 held-out 数据上持续监控
4. BT 模型假设偏好可传递，对存在循环偏好的数据需特殊处理
5. 云 GPU 成本可能快速累积，建议先小规模验证再扩展
