1 问题定义：GRPO 多 Reward 下的 Reward Hacking

Reward Hacking 是指智能体过度拟合到错误指定的奖励模型，优化代理奖励但偏离真实目标的现象。在 GRPO 的多 Reward 设置中，该问题表现为两种典型模式：

模式 A — 高方差奖励主导（MO-GRPO 揭示） GRPO 先聚合所有 Reward 再归一化，导致方差最大的 Reward 主导优势函数。低方差但同样重要的 Reward 被系统性忽略，策略只优化单一目标而牺牲其他目标。
模式 B — 虚假优势（PREF-GRPO 揭示） 点式 Reward 模型对同组相似图像打分过于接近，标准差极小。归一化后，微小分数差异被过度放大，产生巨大但虚假的优势值，导致策略过度更新和奖励欺骗。

2 方法一：MO-GRPO 权重归一化层
MO-GRPO Mitigating Reward Hacking of Group Relative Policy Optimization
arXiv 2509.22047 2025.09
核心洞察

原始 GRPO 的优势函数计算顺序为「先聚合后归一化」，这使得高方差 Reward 分量与优势函数的相关性更强（Theorem 1）。MO-GRPO 将顺序改为「先归一化后聚合」，从根本上平衡各 Reward 的贡献。

优势函数对比
GRPO（原始）：先聚合后归一化
Ag = ½ ∑i=1K Ri(q, og) − meano(∑i=1K Ri(q, o)) ⁄ stdo(∑i=1K Ri(q, o))
MO-GRPO（提出）：先归一化后聚合
AgMO = ∑i=1K ½ Ri(q, og) − meano(Ri(q, o)) ⁄ stdo(Ri(q, o))
核心性质
性质 GRPO MO-GRPO
高方差奖励主导 存在 消除
正仿射变换不变性 不满足 满足
需要手动调权重 是 否（自动重加权）
Reward 不相关时各Reward相关性 σi/σ（不均衡） 1/√K（完美均衡）
实现代码
def MO_GRPO(reward_1, reward_2): combined_scores = [] std_r1 = np.std(reward_1) + 1e-6 std_r2 = np.std(reward_2) + 1e-6 r1_norm = (reward_1 - np.mean(reward_1)) / std_r1 r2_norm = (reward_2 - np.mean(reward_2)) / std_r2 for i in range(len(group_samples)): score = (r1_norm[i] + r2_norm[i]) / np.sqrt(2) combined_scores.append(score) return combined_scores

3 方法二：PREF-GRPO 奖励范式变革层
PREF-GRPO Pairwise Preference Reward for GRPO
复旦 & 腾讯 2025.09
核心洞察

点式 Reward 模型对同组相似图像打分过于接近（标准差极小），归一化后微小差异被过度放大，产生虚假优势。PREF-GRPO 从奖励建模范式层面变革：用成对偏好胜率替代绝对分数。

核心结论 PREF-GRPO 从奖励建模范式层面解决问题：将"最大化绝对分数"转变为"拟合成对偏好"，从根本上增大奖励方差，消除虚假优势的土壤。简单多 Reward 加权组合（HPS+CLIP）仍基于点式分数，无法解决根因。
4 方法三：Adv-GRPO 对抗博弈层
Adv-GRPO Adversarial GRPO with Visual Foundation Models
CVPR 2026 NUS Show Lab & ByteDance
核心洞察

预训练 Reward 模型存在固有偏差（PickScore 降低图像质量，OCR Reward 降低美学），且容易被 hack。Adv-GRPO 将图像生成建模为对抗博弈，用学习型奖励和视觉基础模型提供密集信号。

视觉基础模型作为奖励（DINO）
  • 密集视觉信号：DINO 提供跨多维度的一致性特征，而非单一标量分数
  • DINO 相似度奖励：衡量生成图像与参考图像的视觉一致性
  • 对抗性 DINO 奖励：改善视觉保真度、结构清晰度、纹理丰富度、语义连贯性
  • 额外支持RL 分布迁移 / 风格定制（如迁移到动漫风格域）

选型建议
快速修复 / 通用场景 选择 MO-GRPO：改动最小（仅改变优势函数计算顺序），即插即用，无需额外模型或调参。适用于任何多 Reward 的 GRPO 场景。
图像生成专用 / 追求稳定训练 选择 PREF-GRPO：针对 T2I 虚假优势问题设计，从奖励范式层面解决根因。需配备成对偏好 Reward 模型（如 UnifiedReward-Think）。
追求最强防 Hacking / 需要风格定制 选择 Adv-GRPO：对抗机制从根本上避免被 hack，DINO 提供密集视觉信号。需额外训练判别器 + 准备参考图像集，实现最复杂。
工程落地组合拳 MO-GRPO + KL 惩罚 + 早停：兼顾效果和实现成本，适合生产环境快速部署。MO-GRPO 处理多 Reward 均衡，KL 惩罚限制策略偏离，早停防止过优化。
