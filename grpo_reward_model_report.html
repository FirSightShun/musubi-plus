DanceGRPO：视觉生成大一统强化学习框架

港大 & 字节跳动  |  arXiv:2505.07818  |  GitHub ⭐1.6k  |  项目主页
全量开源 Diffusion + Flow T2I + T2V + I2V

首个将 GRPO 统一应用于视觉生成的框架，同时支持 Diffusion 和 Rectified Flow 两大生成范式，覆盖文生图、文生视频、图生视频、图像编辑四项任务。GRPO 固有的稳定性使其能有效克服 RL-based 视觉生成方法的优化挑战。

🎯 Reward Model

  • HPS-v2.1 偏好 — 人类偏好评分，覆盖 SD/FLUX/Qwen-Image/Wan2.1 [HF]
  • PickScore 偏好 — 人类偏好奖励（FLUX 训练用）
  • CLIP ViT-H-14 对齐 — 文本-图像对齐评分 [HF]
  • VideoReward 视频 — 视频质量评估（HunyuanVideo/SkyReels 用）[HF]
  • Qwen2-VL-2B VLM — 视频评估 [HF]

🧩 支持基础模型

  • SD v1.4 — 文生图（8×H800）
  • FLUX.1-dev — 文生图（16×H800, ~12h）[训练后权重]
  • HunyuanVideo — 文生视频（16/32×H800, VQ↑50-60%）
  • SkyReels-I2V — 图生视频（32×H800）
  • Qwen-Image / Qwen-Image-Edit — 文生图/编辑（8×H800）
  • Wan-2.1 — 文生图（8×H800）

Flow-GRPO：首个基于在线 RL 的 Flow Matching 训练方法

 |  arXiv:2505.05470  |  GitHub  |  项目主页  |  在线Demo
全量开源 Flow Matching 9种Reward

首个将在线策略梯度强化学习整合进 Flow Matching 模型的方法，支持最多 9 种奖励模型的灵活组合。在 SD3.5-M 上实现了组合生成、文本渲染和人类偏好对齐三大任务的显著提升。支持多奖励加权组合训练。

🎯 Reward Model（9种）

  • ImageReward 偏好 — 通用 T2I 偏好，捕捉对齐+保真+安全 [HF]
  • PickScore 偏好 — 人类偏好奖励模型
  • UnifiedReward 偏好 — 多模态统一奖励，SOTA [GitHub]
  • GenEval 对齐 — 组合提示词评估
  • OCR 对齐 — 文本渲染准确率（PaddleOCR）
  • Aesthetic 质量 — CLIP 线性回归器美学评分
  • DeQA 质量 — MLLM 图像质量评估 [GitHub]
  • QwenVL VLM — 提示工程奖励
  • JPEG Compressibility 质量 — 图像大小代理指标

🧩 支持基础模型 & 权重

  • SD3.5-M — 主力基座模型
  • GenEval 任务: SD3.5M-FlowGRPO-GenEval
  • Text Rendering: SD3.5M-FlowGRPO-Text
  • Human Pref: SD3.5M-FlowGRPO-PickScore
  • 多奖励组合示例: pickscore:0.5 + ocr:0.2 + aesthetic:0.3

Adv-GRPO：对抗性奖励机制 — 图像即自身奖励

NUS Show Lab & 字节  |  arXiv:2511.20256  |  GitHub ⭐83  |  项目主页
全量开源 CVPR 2026 对抗性奖励

核心创新：引入对抗性奖励框架，迭代更新奖励模型和生成器，从三个维度改进 T2I 生成——(1) 缓解奖励黑客；(2) 用视觉基础模型（DINO）作为奖励信号；(3) 基于 RL 的分布迁移实现风格定制。

🎯 Reward Model

  • PickScore 偏好 — 传统偏好奖励基线 [HF]
  • DINOv2 对抗 — 视觉基础模型作为奖励，提供丰富视觉先验 [HF]
  • DINO_OCR 对抗 — OCR prompts 训练的 DINO 检查点
  • DINO_GENEVAL 对抗 — GenEval prompts 训练的 DINO 检查点

🧩 支持基础模型

  • SD3 — 主力基座（LoRA 微调 + DeepSpeed Zero2/3）
  • Qwen-Image — 用于参考图像生成
  • 在线Demo: PickScore版 / DINO版

AR-GRPO：自回归图像生成的在线 RL 训练

快手 Klear 团队  |  arXiv:2508.06924  |  GitHub ⭐52  |  HuggingFace
全量开源 自回归 C2I + T2I

首次将 GRPO 在线 RL 应用于自回归图像生成模型（LlamaGen），在 Class-to-Image 和 Text-to-Image 任务上均取得显著改进。使用多维度奖励函数覆盖感知质量、真实感和语义保真度。

🎯 Reward Model（7+种）

  • CLIP ViT-L-14 对齐 — 语义对齐评分
  • HPSv2 偏好 — 人类偏好评分
  • PickScore 偏好 — 人类偏好评分
  • ImageReward 偏好 — 通用偏好奖励
  • Aesthetic Score 质量 — 美学质量评分
  • DeQA 质量 — 图像质量评估（需独立后端）
  • UniReward (UnifiedReward-7b) 偏好 — 统一奖励（需 sglang）
  • Qwen2.5-VL-3B VLM — VLM 奖励计算

🧩 支持基础模型

  • LlamaGen — 自回归图像生成基座
  • Class-to-Image (ImageNet 类条件生成)
  • Text-to-Image (文本条件生成)
  • 评估: FID(C2I), GenEval(T2I), CLIP, HPSv2, PickScore 等

💡 关键洞察与趋势

🏆 UnifiedReward 系列成为新标杆

UnifiedReward 是目前覆盖面最广的统一奖励模型，支持图像/视频的理解与生成评估，并提供 Think（CoT推理）、Flex（个性化）、Edit（编辑）三种变体。已获 NeurIPS 2025 接收，被 Meta/NVIDIA/Apple/字节/腾讯/快手等广泛引用，堪称视觉领域 Reward Model 的「瑞士军刀」。

⚔️ 从逐点评分到成对偏好

传统 GRPO 使用逐点评分（Pointwise Score），容易导致 reward hacking。Pref-GRPO 创新性地提出成对偏好奖励（Pairwise Preference Reward），用 win rate 替代绝对分数，训练更稳定。这一思路已被阿里巴巴和通义实验室独立验证。

🔄 对抗性奖励：图像即自身奖励

Adv-GRPO 提出用视觉基础模型（DINOv2）作为奖励信号，而非传统偏好模型。通过对抗性训练迭代更新奖励模型和生成器，从根本上缓解 reward hacking。这种「图像即自身奖励」的范式为视觉生成 RL 提供了新思路。

🧩 多奖励组合成为标配

几乎所有 GRPO 变体都支持多奖励加权组合训练。典型组合：偏好奖励（PickScore/HPSv2）+ 质量奖励（Aesthetic/DeQA）+ 对齐奖励（CLIP/OCR）。Flow-GRPO 支持 9 种奖励灵活配置，DanceGRPO 也通过 Issue #19 提供多奖励训练代码。

📈 VLM 作为 Reward 正在兴起

从 Qwen2-VL、Qwen2.5-VL 到 UnifiedReward（基于 Qwen3-VL），视觉语言模型（VLM）作为 Reward Model 成为明确趋势。VLM 天然具备文本-图像理解能力，且可通过 prompt engineering 灵活定义奖励维度，是传统 CLIP-based 方法的有力替代。

🚀 开源生态日趋完善

5 个 GRPO 变体全部开源了完整代码+模型权重，DanceGRPO 和 Flow-GRPO 还提供 HuggingFace Spaces 在线 Demo。从代码框架（基于 diffusers/FastVideo）到训练脚本、评估基准、预训练权重，形成了从研究到落地的完整闭环。
